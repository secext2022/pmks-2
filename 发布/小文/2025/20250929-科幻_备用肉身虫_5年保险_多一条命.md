# (科幻) 备用肉身虫 (5 年保险, 多一条命)

所有基于 "纯机器" 的长期生命维持系统, 都失败了. 机器和肉身的兼容性太差, 实现成本太高 (每日使用成本高达几十万元), 并且 BUG 太多, 容易出现一大堆问题, 并且坚持不了多长时间 (无法长期连续使用).

所以, 基于肉身 "兼容层" 的高级生命维持系统, 出现了 ! 系统结构为: 机器 -> 兼容层 -> 使用者. 从人体获取一些细胞, 进行体外的培养与改造, 最终形成 "一条大虫子" 模样的 "兼容层".

"大虫子" 包含多个关键生理系统, 包括 "肝脏" 类似物, 呼吸系统, 消化系统, 循环系统, 泌尿系统, 等. 由于伦理问题, "大虫子" 不含神经细胞. 需要神经调控的部位, 使用植入型 "微电极" 驱动, 也就是直接使用芯片控制.

"大虫子" 对接一套 "机器" 生命维持系统, 但是 "大虫子" 和 "机器" 的复杂度都显著降低. 由于 "大虫子" 无需实现独立生存,各大系统都可以简化. 比如消化系统无需口腔, 胃等结构, 只需直接吸收配置好的营养液即可. 比如呼吸系统无需鼻腔/口等加湿/加热/滤尘结构, 机器会做这些, 只需通气孔连接机器的管道, 实现气体交换功能即可. 并且参考鸟肺的结构设计, 气体交换有 "进口" 和 "出口", 实现连续气体交换, 显著增加呼吸效率, 降低 "肺" 的体积和重量. 比如泌尿系统只需 "肾" 而无需 "膀胱".

"机器" 这边也简化了设计, 由于无需直接对接复杂的生理系统, 只需提供一些简单的外部条件, 比如营养液, 干净的空气, 进行温度/湿度调节, 基于程序的简单芯片监控及调节 (替代 "神经系统"), 等.

由于免疫排斥, 每个人都有自己的 "秘钥", 所以 "机器" 部分可以通用, 但 "大虫子" 兼容层只能专人专用. 所以, 每个想使用这套系统的人, 都要提取细胞, 培养自己的 "大虫子".

"大虫子" 从开始制造到能够正常使用, 需要一年的时间, 最终大虫子的重量需要达到用户体重的 30%, 也就是大约 20 ~ 30kg. 临时抱佛脚肯定来不及, 所以要提前备用.

好在 "大虫子" 技术普及之后, 成本并不高, 新培养一条只需几千元, 设计寿命 5 年, 每年的保存维护费用也只有几千元.

这就相当于, 每年花几千元购买一份 "保险", 培养一条 "大虫子" 保存在用户常住城市的医院里. 一旦自己严重受伤或疾病, 这条 "大虫子" 可以让自己多一条命.

即使多器官系统严重衰竭, 只需把血管接上 "大虫子", 即可 "瞬间 满血复活" (误, 划掉) .. . 咳咳, 即可继续维持生命. 每天的使用费用只需百元级别. 最差的情况, 全身都坏掉了, 只剩一颗头 (大脑), 那么将血管连接至 "大虫子", 即可长期维持 "头" 的正常存活. 一条 "大虫子" 最多可连续使用 2 年, 到期后可更换新的 "大虫子", 继续续命.

之后怎么办, 难道余生就要一直这样 "人头虫身" 了嘛 ? 不 ! 制造大虫子的技术, 同样能用来再造一副躯体 ! 即使只剩一个头, 大约经过 5 年时间, 就能再造一个 "身体" 出来, 到时侯头有了新身体的支持, 就能脱离 "大虫子", 正常生活了.

期间, 连接 "大虫子" 的血管就像新的 "脐带", 用户也能字面意思上的 "重获新生" !

设计灵感来源: WSL, 虚拟机

WSL 1 试图直接使用 API 转换来兼容 Linux, 但是工程上失败了, BUG 太多.

WSL 2 通过在虚拟机运行 Linux 内核, 几乎实现了完美兼容.

不同操作系统同为计算机上运行的软件, 硬件完全相同, 并且是人造物, 要实现 API 兼容都如此困难. 而 "生理系统" 和 "机器" 的差异如此巨大, "长期生命维持系统" 的难度可想而知. 所以, 加入 "大虫子" 中间兼容层, 或许可以显著降低工程难度, 实现良好的效果.
